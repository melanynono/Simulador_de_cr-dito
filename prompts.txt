3. PROMPT PARA HACER EL DISEÑO RESPONSIVE
================================================================

"Implementa un diseño completamente responsive para el simulador de crédito con los siguientes breakpoints:

BREAKPOINT 1: DESKTOP (>768px) [Por defecto]
- Layout de 2 columnas en formularios
- Nav horizontal en header
- Hero con título grande (2.5rem)
- Cards con padding generoso (2rem)
- Grids con minmax(250px, 1fr) para auto-ajuste

BREAKPOINT 2: TABLETS (≤768px)
- Header cambia a layout vertical centrado
- Logo y brand info centrados
- Navegación wrap horizontal centrada
- Hero title reduce a 2rem
- Formularios cambian a 1 columna
- Results grid a 1 columna
- Botones apilados verticalmente
- Trust section a 1 columna
- Padding reducido en cards (1.5rem)

BREAKPOINT 3: MÓVILES (≤480px)
- Variables CSS ajustadas para espaciado reducido
- Logo pequeño (40x40px)
- Brand name reduce a 1.25rem
- Nav links más compactos (0.875rem)
- Hero padding reducido
- Hero title a 1.5rem
- Main wrapper con padding mínimo
- Cards con padding 1rem
- Section icons 40x40px
- Form inputs con padding reducido (0.75rem)
- Result values con tamaños más pequeños
- Botones con texto 0.875rem
- Trust icons 2rem

BREAKPOINT 4: MÓVILES PEQUEÑOS (≤320px)
- Logo section con gap mínimo
- Brand name a 1.125rem
- Hero title a 1.25rem
- Section header en columna y centrado
- Ajustes finales de espaciado

TÉCNICAS RESPONSIVE:
- Grid con auto-fit para adaptación automática
- Minmax() para anchos flexibles
- Media queries en cascada de mayor a menor
- Sin overflow horizontal garantizado
- Inputs con width 100% y box-sizing border-box
- Flex-wrap en navegación
- Variables CSS que se ajustan por breakpoint

PRUEBAS:
- Verificar en Chrome DevTools
- Probar en iPhone SE (320px)
- Probar en tablets (768px)
- Probar en desktop (1200px+)
- Sin scroll horizontal en ningún breakpoint
- Touch targets mínimo 44x44px en móvil
- Textos legibles sin zoom"
